# GPX Repair

Автоматически исправляет GPX треки которые попортила глушилка. Реальные данные не восстанавливает, но находит неадекватный участок, убирает его и плавно соединяет начало и конец глючного участка.

## Веб-версия

**https://agran.github.io/gpx-repair/**

Загрузите GPX файл прямо в браузере — никаких установок, никакого сервера.

## Python-версия

```bash
python fix_gpx_jammer.py input.gpx
python fix_gpx_jammer.py input.gpx output.gpx
```

### Параметры

| Параметр            | По умолчанию | Описание                                                        |
| ------------------- | ------------ | --------------------------------------------------------------- |
| `--max-speed`       | 20 м/с       | Максимальная допустимая скорость, при превышении — телепортация |
| `--min-distance`    | 1000 м       | Минимальное расстояние аномального прыжка                       |
| `--pre-jitter-dist` | 200 м        | Убирать точки дрейфа перед основным прыжком                     |
| `--max-vert-speed`  | 5 м/с        | Максимальная допустимая вертикальная скорость                   |
| `--interval`        | 1 с          | Шаг интерполяции между точками                                  |
| `--no-interpolate`  | —            | Не заполнять пробел, просто удалить                             |
| `--gap-fill`        | из профиля   | Способ заполнения пробела: `line` — прямая линия (офлайн), `foot`/`bike`/`car` — маршрут по дорогам и тропам через OSRM (routing.openstreetmap.de) с высотой рельефа из Open-Meteo. Требует интернет; при ошибке сети молча откатывается на `line` |

## Как это работает

1. Находит точки где скорость между соседними координатами физически невозможна (телепортация)
2. Определяет весь проблемный участок — от первого дрейфа координат до возврата к нормальной траектории
3. Дополнительно обрезает аномальные скачки высоты на границах участка
4. Удаляет проблемный участок и заполняет пробел прямолинейной интерполяцией
