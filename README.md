# GPX Repair

Автоматически исправляет GPX треки которые попортила глушилка. Исходный (реальный) маршрут на глючном участке не восстанавливает, но находит неадекватный участок, убирает его и заполняет пробел между началом и концом глючного участка — либо плавной кривой (на стыках она идёт по направлению реального трека, а не по прямой с изломом), либо (для большинства профилей активности — по умолчанию) правдоподобным маршрутом по реальным дорогам/тропам с высотой рельефа.

## Веб-версия

**https://agran.github.io/gpx-repair/**

Загрузите GPX файл прямо в браузере — никаких установок, никакого сервера.

## Python-версия

```bash
python fix_gpx_jammer.py input.gpx
python fix_gpx_jammer.py input.gpx output.gpx
```

### Параметры

| Параметр            | По умолчанию | Описание                                                                                                                                                                                                                                                 |
| ------------------- | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--profile`         | `hiking`     | Профиль активности — задаёт значения параметров ниже по умолчанию: `hiking`, `running`, `kayak`, `horse`, `mtb`, `road_bike`, `ski`, `enduro`, `boat`, `car`, `paraglider`, `train`. Явно указанные флаги ниже всегда переопределяют значение профиля    |
| `--max-speed`       | из профиля   | Максимальная допустимая скорость, при превышении — телепортация                                                                                                                                                                                          |
| `--min-distance`    | из профиля   | Минимальное расстояние аномального прыжка                                                                                                                                                                                                                |
| `--pre-jitter-dist` | из профиля   | Убирать точки дрейфа перед основным прыжком                                                                                                                                                                                                              |
| `--max-vert-speed`  | из профиля   | Максимальная допустимая вертикальная скорость                                                                                                                                                                                                            |
| `--interval`        | из профиля   | Шаг интерполяции между точками                                                                                                                                                                                                                           |
| `--no-interpolate`  | —            | Не заполнять пробел, просто удалить                                                                                                                                                                                                                      |
| `--gap-fill`        | из профиля   | Способ заполнения пробела: `line` — простое соединение (офлайн), `foot`/`bike`/`car` — маршрут по дорогам и тропам через OSRM (routing.openstreetmap.de) с высотой рельефа из Open-Meteo. Требует интернет; при ошибке сети молча откатывается на `line` |
| `--quiet`, `-q`     | —            | Не выводить подробный отчёт                                                                                                                                                                                                                              |

## Как это работает

1. Находит точки где скорость между соседними координатами физически невозможна (телепортация)
2. Определяет весь проблемный участок — от первого дрейфа координат до возврата к нормальной траектории
3. Дополнительно обрезает аномальные скачки высоты на границах участка
4. Удаляет проблемный участок и заполняет пробел — прямой линией либо маршрутом по дорогам/тропам (`--gap-fill`)
